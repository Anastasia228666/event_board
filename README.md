Event Board - це веб-додаток з використанням класичного стеку Java 
EE (Servlets) та реляційної бази даних. Архітектура побудована за класичною тришаровою схемою: модель, репозиторій, сервіс та контролер.

Основою бізнес-логіки є сутність Event із полями для ID, назви, дати, ліміту та залишку місць.
Взаємодія з базою даних PostgreSQL реалізована в EventRepository за допомогою чистого JDBC. Метод findAll() через SQL-підзапит до таблиці учасників динамічно вираховує кількість вільних місць, а saveEvent() безпечно додає нові події через PreparedStatement.

Клас EventService містить бізнес-логіку та за допомогою Stream API перевіряє можливість реєстрації на захід.
Обробку HTTP-запитів виконує EventServlet (/events): GET завантажує список подій, а POST приймає дані з форми й створює новий запис.
Інтерфейс користувача реалізовано в index.jsp, де через JSTL-теги відображається таблиця заходів та форма додавання нових подій. У pom.xml, окрім драйвера БД та сервлетів, також підключено JUnit і Mockito для тестування.
