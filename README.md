# event_board


База даних

1.Відкрити консоль своєї бази даних.
2.Виконати SQL-скрипт для створення таблиці.
3.У файлі EventRepository.java створити свої змінні url, user та password
  і вписати свої власні дані підключення.

 Налаштування Tomcat

 1.У верхній панелі IDEA натиснути Edit Configuration.
 2.Натиснути + -> Tomcat Server -> Local.
 3.Перейти у вкладку Deployment, натиснути + -> Artifact.
 4.Обрати артефакт EventBoard:war exploded.
 5.Натиснути OK, а потім зелений трикутник (Run) для запуску сервера.

 (+ доп. не обов'язково читати)

 Структура проєкту

 Код розділений на логічні пакети в папці src/main/java:

 1.model: Клас Event - опис об'єкта заходу (id, назва, дата, місця).
 2.repository: Клас EventRepository - логіка підключення до БД та SQL-запити (Select/Insert).
 3.service: Клас EventService - бізнес-логіка (перевірка вільних місць).
 4.controller: Клас EventServlet - обробка HTTP-запитів (doGet/doPost) та перенаправлення на сторінку.
 5.webapp: Файл index.jsp - візуальний інтерфейс користувача (таблиця заходів та форма додавання).
